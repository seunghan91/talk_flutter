# Talkk - 앱 워크플로우

> 음성으로 연결되는 새로운 소셜 네트워크

---

## 1. 앱 개요

| 항목 | 내용 |
|------|------|
| **앱 이름** | Talkk (톡크) |
| **핵심 컨셉** | 음성 기반 소셜 네트워크 |
| **핵심 기능** | 음성 브로드캐스트 → 답장 → 1:1 대화 |
| **플랫폼** | iOS / Android (Flutter) |
| **API 서버** | Rails 7.1 (talkk-api.onrender.com) |
| **주요 특징** | 6일 자동 만료, 지능형 수신자 매칭 |

     ▼
  저장된 토큰 확인 ──── 있음 ────▶ [홈 화면]
     │
     없음
     │
     ▼
  ┌─────────────┐
  │  로그인 화면  │
  └──────┬──────┘
         │
    ┌────┴────┐

    │         │
    │         ▼
    │    ┌──────────────┐
    │    │ 전화번호 입력  │
    │    └──────┬───────┘
    │           ▼
    │    ┌──────────────┐
    │    │ SMS 인증코드   │
    │    │   발송 & 확인  │
    │    └──────┬───────┘
    │           ▼
    │    ┌──────────────┐
    │    │ 프로필 설정    │
    │    │ (닉네임/성별)  │
    │    └──────┬───────┘
    │           │
    ▼           ▼
  ┌─────────────────┐
  │  JWT 토큰 발급   │
  │  로컬 보안 저장   │
  └────────┬────────┘

```

### API 엔드포인트

| 단계 | API | 설명 |
|------|-----|------|
| SMS 요청 | `POST /auth/phone-verifications` | 인증번호 발송 |
| SMS 확인 | `POST /auth/phone-verifications/verify` | 코드 검증 |
| 회원가입 | `POST /auth/registrations` | 계정 생성 |
| 로그인 | `POST /auth/sessions` | 토큰 발급 |

## 4. 핵심 기능: 음성 브로드캐스트

### 4-1. 브로드캐스트 생성 (발신)

```
┌─────────────────────────────────────────────────────────────┐
│                  브로드캐스트 생성 플로우                       │
└─────────────────────────────────────────────────────────────┘

  [홈 화면]
     │
     │  FAB (마이크 버튼) 탭
     ▼
  ┌──────────────────┐
  │   녹음 화면       │
  │                   │
  │  ┌─────────────┐  │
  │  │  파형 시각화  │  │    ◀── 실시간 오디오 파형
  │  └─────────────┘  │
  │                   │
  │    00:00 / 01:00  │    ◀── 최대 60초
  │                   │
  │    ◉ [녹음 버튼]   │    ◀── 탭: 시작/중지
  │                   │
  │  [취소]   [전송]   │
  └──────────────────┘
         │
         │ 녹음 완료 & 전송
         ▼
  ┌──────────────────┐
  │  서버 업로드       │
  │                   │
  │  • 음성 파일 업로드 │
  │  • 수신자 자동 선택 │    ◀── 알고리즘 기반 5명 매칭
  │  • 6일 만료 설정   │
  └──────────────────┘
         │
         ▼
  수신자 5명에게 푸시 알림 발송
```

### 4-2. 브로드캐스트 수신 & 답장

```
┌─────────────────────────────────────────────────────────────┐
│                브로드캐스트 수신 → 답장 → 대화 시작              │
└─────────────────────────────────────────────────────────────┘

  [수신자 A의 홈 화면]
     │
     │  새 브로드캐스트 수신 (피드에 표시)
     ▼
  ┌────────────────────────────┐
  │  김채현님의 브로드캐스트       │
  │  ┌────────────────────┐    │
  │  │ ▶ ━━━━━━━━━━ 0:45  │    │    ◀── 음성 재생 (파형)
  │  └────────────────────┘    │
  │                             │
  │  ⏰ 만료까지 5일 남음         │
  │                             │
  │          [답장하기 ▶]        │
  └─────────────┬──────────────┘
                │
                │  "답장하기" 탭
                ▼
  ┌────────────────────────────┐
  │  답장 녹음 화면              │
  │                             │
  │  ◉ [녹음] → [전송]          │
  └─────────────┬──────────────┘
                │
                │  답장 전송
                ▼
  ┌────────────────────────────┐
  │  1:1 대화 자동 생성          │
  │                             │
  │  메시지 화면에 새 대화 등장!   │
  │  ┌────────────────────┐    │
  │  │ 김채현  •  방금 전    │    │
  │  │ 🎤 음성 메시지 0:45  │    │
  │  └────────────────────┘    │
  └────────────────────────────┘
```

## 7. 알림 시스템

```
┌─────────────────────────────────────────────────────────────┐
│                      알림 (Push Notification)                 │
└─────────────────────────────────────────────────────────────┘

  [이벤트 발생]
       │
       ├── 새 브로드캐스트 수신 ──▶ "김채현님이 음성을 보냈어요"
       │
       ├── 브로드캐스트 답장 ────▶ "이지원님이 답장했어요"
       │
       ├── 새 메시지 수신 ──────▶ "박지민님이 음성 메시지를 보냈어요"
       │
       └── 시스템 공지 ─────────▶ "Talkk 업데이트 안내"

       │
       ▼
  ┌──────────────┐
  │  FCM 푸시 발송 │
  └──────┬───────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
  앱 포그라운드    앱 백그라운드
  (인앱 알림)     (시스템 알림)
    │                │
    ▼                ▼

  • 브로드캐스트 → 방송 상세 화면

| 설정 | 설명 | 기본값 |
|------|------|--------|
| 전체 알림 | 모든 푸시 알림 ON/OFF | ON |
| 브로드캐스트 알림 | 새 방송 수신 알림 | ON |
| 메시지 알림 | 새 메시지/답장 알림 | ON |

## 10. 핵심 사용자 시나리오

### 시나리오 1: 첫 사용자의 여정

```
1. 앱 설치 & 실행
2. 회원가입 (전화번호 인증 → 닉네임/성별 설정)
3. 홈 화면 진입 → 아직 받은 브로드캐스트 없음
4. FAB(마이크) 탭 → 60초 이내 음성 녹음
5. 전송 → 알고리즘이 5명의 수신자에게 전달
6. 대기...
7. 누군가 답장! → 메시지 탭에 새 대화 생성
8. 1:1 음성 대화 시작
```

### 시나리오 2: 브로드캐스트 수신자의 여정

```
1. 푸시 알림: "김채현님이 음성을 보냈어요"
2. 앱 열기 → 홈 피드에 새 브로드캐스트 표시
3. 재생 버튼 탭 → 음성 듣기
4. 마음에 들면 "답장하기" 탭
5. 음성 답장 녹음 → 전송
6. 1:1 대화 자동 생성 → 메시지 탭으로 이동
7. 음성 메시지 주고 받으며 대화
```

### 시나리오 3: 활발한 사용자의 일상

```
1. 아침: 앱 열기 → 새로운 브로드캐스트 확인 & 듣기
2. 마음에 드는 방송에 답장
3. 메시지 탭에서 기존 대화 확인 & 답장
4. 즐겨찾기(★)한 대화 우선 확인
5. 저녁: 새 브로드캐스트 녹음 & 전송
6. 알림으로 답장 확인
```
